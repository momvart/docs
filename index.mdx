Leaf is a concolic execution framework for Rust that works by instrumenting your program at the MIR level through a custom compiler wrapper called `leafc`. At runtime, a pluggable backend receives callbacks from the instrumented binary and performs analyses ranging from symbolic execution to control-flow tracing — without requiring you to modify your program's logic.

## Get up and running in five steps

<Steps>
  <Step title="Install leafc">
    Clone the repository and install the Leaf compiler wrapper with Cargo.

    ```bash
    git clone https://github.com/sfu-rsl/leaf.git && cd leaf
    cargo install --path ./compiler
    ```
  </Step>
  <Step title="Build a runtime backend">
    Compile the symbolic execution backend (or another backend of your choice) as a shared library.

    ```bash
    cargo build -p runtime_symex
    ```
  </Step>
  <Step title="Link the shared library">
    Create the expected directory, symlink the compiled library into it, and export the path so instrumented binaries can find it at startup.

    ```bash
    mkdir -p target/debug/runtime_symex
    ln -sf "$(find target/debug -maxdepth 1 -name 'libleafrt*.so' | head -n 1)" \
        target/debug/runtime_symex/libleafrt.so
    export LD_LIBRARY_PATH="$PWD/target/debug/runtime_symex:$LD_LIBRARY_PATH"
    ```
  </Step>
  <Step title="Compile your program with leafc">
    Use `leafc` in place of `rustc` to compile and instrument a target program.

    ```bash
    leafc samples/hello_world.rs
    ```
  </Step>
  <Step title="Run the instrumented binary">
    Enable runtime logging, then execute the binary — the backend produces analysis output automatically.

    ```bash
    export LEAF_LOG="info"
    ./hello_world
    ```
  </Step>
</Steps>
